# Drupal Tailwind

This project integrates the [Tailwind CSS](https://tailwindcss.com) into Drupal by providing a module that manages the Tailwind and compiles CSS directly on the server.
There are no external Node dependencies, that you need to install or setup. This project uses the standalone Tailwind CLI tool.

## Getting started
1. Download the module and place it in your `modules/` directory.
2. Enable the module through Drupal UI or Drush.
3. Configure the module at `/admin/config/development/styles/tailwind` to select your theme and other settings.
4. Download the Tailwind CLI binary using the **Download compiler runnable** button on the settings page. The runnable that will be downloaded is the official runnable created by the Tailwind team. You do not have to choose your OS, module does that automatically. (Supported: Linux, macOS, Windows)
5. Once the download is finished, and you've set up the module, you can configure the extensions per theme and module to avoid scanning unnecessary files. (All extensions are disabled by default.)
6. If you have done everything mentioned above, you are all set.

## How does it work?
Since the dependency hell of Node.js is pain for the project setup, this module uses the CLI version, which eliminates the need for Node.js entirely.
Tailwind compiles on each page load automatically by default, if any files have changed in the configured theme or modules.
And that's pretty much it. There are some options you may want to use, so if ticked they are passed to the Tailwind CLI. (minify, optimize, map etc.)

## Including the compiled CSS in your theme

Add the compiled stylesheet to your theme's `.libraries.yml`:

```yaml
global:
  css:
    theme:
      dist/css/tailwind.css: {}
```

Then attach the library in your theme's `.info.yml`:

```yaml
libraries:
  - your_theme/global
```

## Requirements

- Drupal ^10|^11
- Linux, macOS, or Windows

## Things to keep in mind
- The module does not provide a Tailwind theme for now
- Tailwind v4+ uses layers, meaning its easy to override
- You have to include the css from '/dist/css/' folder in your theme's libraries.yml and info.yml
- The runnable that is downloaded should not see the light of git, so please make sure to exclude the bin folder from your git repo.
- Files are autogenerated in the theme's tailwind folder, so please do not edit them. The only file you should edit is the `theme.css` file, which is preserved across recompilations.

I believe that this setup is a lot easier than the Node.js setup, and it removes obscurity in the development process especially when working with multiple devs.
